The top 3 popular hour of day to submit is 18h, 23h, and 13h. It makes sense because they to submit so that they could go eat lunch/dinner, and 23h is just before the deadline. I’m surprised that 23h isn’t the most popular.

We can also produce one histogram for every student:

ggplot(cs1010s, aes(x = hour(Submitted))) + geom_histogram(binwidth = 1) + facet_wrap(~Name)

I discovered that CZ and WW really like to submit at 23h, and CY and LY like to submit just before dinner. LY submitted more than one third of his assignments past midnight!

To account for times later than 2359, I used the Japanese extended hour system: 0 to 5 a.m. are called 24h to 29h in Japan.

hour_ext <- function(dttm) {
  h <- as.integer(format(dttm, format = "%H"))
  m <- as.integer(format(dttm, format = "%M"))
  if(h < 7) {
    h <- h + 24
  }
  return(h + m / 60)
}

hour_ext <- Vectorize(hour_ext)

Let’s see if our function works.

ggplot(cs1010s, aes(x = hour_ext(Submitted))) + geom_histogram(binwidth = 0.5)

Nice. Now let’s see the student’s distribution as a box plot.

ggplot(cs1010s, aes(x = hour_ext(Submitted), y = Name)) + geom_boxplot()

ZY submits her assignments the earliest in the day, with median around 13h, while LZ’s median being the latest, which is more than 22h. SA has the widest range, she could submit anytime from 10h to 27h.

7 Tutor’s takeway

When a student submits an assignment, he/she waits for some time for me to grade it. This period of time is called grading time. The average of grading time is ranked among all 55 tutors. My average grading time is currently 4 hours flat. However, that comes at a cost of visiting the website several times a day.

Moving on, I want to visit coursemology just two times a day. How do I minimize my average grading time, assuming the students’ habits remain constant?

grading_time <- function(submt, grade) {
  ifelse(submt <= grade, grade - submt, grade + 24 - submt)
}

grading_time <- Vectorize(grading_time)

mean_gt <- function(timestamps, t1, t2) {
  gt1 <- grading_time(timestamps, t1)
  gt2 <- grading_time(timestamps, t2)
  return(mean(pmin(gt1, gt2)))
}

The function mean_gt calculates the mean grading time of all assignments if I check the website at t1 and t2 hours. For example:

submission_times = cs1010s$h_e
mean_gt(submission_times, 12, 24)
## [1] 6.021242

This tells me that if I check everyday at noon and midnight, the students will have to wait 6.0 hours on average. I ran a loop to find the best time combinations, considering I wake up at 10h and go to sleep at 25h.

time1 <- seq(10, 25, 0.1)
time2 <- seq(10, 25, 0.1)

min_gt <- 24
min_t  <- 0

for (i in time1) {
  for (j in time2) {
    gt <- mean_gt(submission_times, i, j)
    if (gt < min_gt) {
      min_gt <- gt
      min_t <- c(i, j)
    }
  }
}

min_gt
## [1] 4.196405
min_t
## [1] 18.3 25.0

If I go grade the assignments at 18.3h and 25.0h, I’d expect my average grading time to be 4.20 hours. Not too bad!
